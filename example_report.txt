## 6. SECURITY CONCERNS & ISSUES

### 🔴 CRITICAL ISSUES

1. **Weak Certificates - Self-Signed "Snakeoil"**
   - Files: `/etc/ssl/certs/ssl-cert-snakeoil.pem` and `/etc/ssl/private/ssl-cert-snakeoil.key`
   - Impact: All TLS connections will show certificate warnings/errors
   - Should be: Valid, signed certificate (Let's Encrypt/CA signed)

2. **Plaintext Authentication Not Enforced in Dovecot**
   - `disable_plaintext_auth` is commented out (default: disabled)
   - Users can authenticate with plaintext over unencrypted connections
   - Recommendation: Set `disable_plaintext_auth = yes` in `/etc/dovecot/conf.d/10-auth.conf`

3. **No TLS Version Minimum in Dovecot**
   - `ssl_min_protocol = TLSv1.2` is commented out
   - Potentially allows SSLv3/TLS 1.0/1.1 (deprecated, vulnerable protocols)
   - Recommendation: Uncomment and enforce TLS 1.2+

### 🟠 HIGH PRIORITY ISSUES

4. **VRFY Command Enabled (User Enumeration)**
   - Postfix `disable_vrfy_command = no`
   - Allows remote enumeration of valid email addresses
   - Recommendation: Set `disable_vrfy_command = yes` in `/etc/postfix/main.cf`

5. **SMTP Smuggling Risk - Bare Newlines Not Forbidden**
   - `smtpd_forbid_bare_newline = no` (CVE-2023-51764 mitigation disabled)
   - Recommendation: Set to `normalize` or `yes` in `/etc/postfix/main.cf`
   - Note: Postfix has patches for this, but setting disabled

6. **No HELO/EHLO Validation**
   - `smtpd_helo_required = no`
   - Allows SMTP clients to proceed without proper greeting
   - Recommendation: Set `smtpd_helo_required = yes`

7. **No Anti-Spam/Bot Protection (Postscreen)**
   - Postscreen is commented out in `/etc/postfix/master.cf`
   - Recommendation: Enable for:
     - Connection rate limiting
     - DNS blocklist checking
     - SPF/DKIM validation
     - Temporary blacklisting (greylisting)

### 🟡 MEDIUM PRIORITY ISSUES

8. **Submission Port (587) Not Enabled**
   - Submission service is defined but not active
   - Forces clients to use unencrypted port 25 or SMTPS
   - Recommendation: Uncomment submission listener in `/etc/postfix/master.cf`

9. **SMTPS (465) Not Enabled**
   - No explicit TLS wrapper mode for SMTP
   - Recommendation: Enable for legacy client support

10. **mbox Mail Format (Legacy)**
    - `mail_location = mbox:~/mail:INBOX=/var/mail/%u`
    - mbox is outdated; Maildir or mdbox preferred
    - Impact: Slower access, single file corruption risk
    - Recommendation: Migrate to Maildir format

11. **Only IMAP Enabled (No POP3)**
    - POP3 support defined but disabled
    - Modern approach, but worth auditing if clients need it

12. **Hostname Configuration Issues**
    - `myhostname = localhost` (should be FQDN)
    - Configured as: `mail.example.com` in mydestination
    - `/etc/mailname = mail.example.com`
    - Recommendation: Set `myhostname = mail.example.com` in `/etc/postfix/main.cf`
